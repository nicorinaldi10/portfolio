What is the Integration of an ABB IRB120 with ROS?

The Integration of an ABB IRB120 with ROS was a project focused on enabling the use of the ABB IRB120 robot with the Robot Operating System (ROS). Initially, the robot could only be programmed using RobotStudio or the RAPID high-level language, both proprietary tools from ABB that were not part of the school's curriculum.

The goal was to develop a tool that connected the robot to ROS, allowing users to:

  • Visualize its manual movement, controlled by an operator.
  • Control the robot directly from ROS applications.

To achieve this, a socket connection using the TCP/IP protocol was implemented, managed with Python and RAPID in RobotStudio. On the ROS side, Gazebo and RVIZ were used to create a virtual model of the robot, simulate its movement, and plan trajectories while avoiding obstacles.

Ultimately, the project aimed to serve as an educational tool, integrating the IRB120 into ROS-based learning and expanding its usability. It allowed users without prior knowledge of RAPID to execute movement programs and develop applications without requiring costly software licenses.



RQT Graph: Node Communication in ROS

Figure below shows the RQT graph of the application, illustrating the communication between different ROS nodes.

The /robot_state_publisher node transmits the robot's current position via the /tf topic to both /rviz_nico_VirtualBox_... and /move_group. This allows MoveIt! to be aware of the robot’s position within the simulation.

The /move_group node also receives information from /rviz_nico_VirtualBox_... regarding potential collisions with objects in the environment. This data is essential for planning safe trajectories and avoiding obstacles during motion execution.

Once a movement is planned, /move_group sends the motion commands to both /rviz_nico_VirtualBox_... for visualization and to /gazebo to execute the movement within the simulation.

Additionally, /gazebo acts as the main simulation node, sending clock signals to all connected nodes to synchronize their operations, ensuring smooth execution of the planned movements.

This structure enables real-time motion visualization and accurate trajectory planning, integrating Gazebo for simulation and MoveIt! for motion control.
