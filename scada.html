What is an SCADA application of an Industrial Elevator?

The application aims to control the operation of the elevator, monitor the status of variables at all times, notify unwanted or abnormal situations through alarms, and store these notifications in a historical log. Additionally, it assigns each user the appropriate permissions according to their responsibilities within the process.


Application Components:

The application consists of ten windows (four of them equivalent but more or less restrictive depending on the user's rank):

  • Home
  • Access Control
  • Administrator, Engineer, Operator, None
  • Synoptic
  • Alarms
  • Alarm Dictionary
  • Historical Trend

The functionality of each window will be explained in detail below.


Home:

This first window consists of two headers displaying the application's title and authorship, accompanied by an image of the university where it was developed.

Below, there is a brief manual with basic instructions for its operation, along with a box that continuously displays the date and time (this will appear on all windows, so it will not be mentioned further).

The interactive Access Control button completes the window; when pressed, it directly leads to the next screen, where the user can register.



Access Control:

The 'Access Control' window allows users to log in by entering their credentials: a username ('User') and password ('Password'). Once both fields are completed, one of the three interactive buttons on the right side will become enabled, granting access to the next window.

To log out of the session, simply click the 'Logout' button.




User windows “Administrator”, ”Engineer” ,”Operator” y ”None”:

Each button in the "Access Control" window leads to a new user window. The only difference between these windows is the level of restriction they present. Therefore, the "Administrator" window can be taken as a reference, extrapolating its functionality to the other three windows (since it does not make sense to explain the functions of the same buttons three times).

The "Administrator" window has four buttons, each providing access to another window that manages a part of the plant supervision process. A specific access level is required for each user window:



Administrator (9999) > Engineer (8500) > Operator (5000) > None (0)



This level is assigned automatically upon registration.



The buttons in question are:

  • SYNOPTIC: redirects to the window supervising how the process is carried out in real-time, including sensors, actuators, and even an animated schematic continuously displaying the plant's current status.
  • ALARMS: redirects to the window notifying undesirable situations.
  • HISTORICAL TREND: opens a window chronologically collecting the previously mentioned notifications, storing them in a graphical or textual document.
  • Access Control: previously explained.

In the other user windows, buttons progressively disappear as the access level decreases, down to "None" (access level = 0), where users only have access to the date and time and the registration panel.




Window SYNOPTIC:

This window is divided into three control panels and a screen that shows the process sequence in real time.

The first panel (Actuators) displays the switches corresponding to these field devices. Additionally, it allows the user to interact with and manage their status.




The second panel (Sensors) only allows monitoring without the possibility of modifying their status from the control center. It accurately represents the real state of the sensors, whether they use inverse logic or not.

The third panel contains only the "Emergency Stop" switch, which completely halts the cycle by turning off all actuators.

The window is complemented by a schematic of the plant, enabling real-time graphical monitoring of the process status (see the brief attached video).

This is possible thanks to a small script that coordinates the horizontal and vertical movements of the container, as well as the vertical-only movement for the elevator. The script considers these three variables based on which switches (actuators) are active at any given time.



Window ALARMS:

The "ALARMS" window allows the reception of alerts for undesired or erroneous situations in the plant. These alerts are indicated through red lights located below the alarm identifier.




Before viewing the panels in operation, it is important to mention the “ALARM” button. This button enables alerts for undesired or erroneous situations in the plant. These situations are indicated by red lights appearing below each alarm identifier.

All alarms are of the "Memory" type with priorities from 1 to 999, except for the analog alarm, which is of the "Memory Integer" type.




Before observing the panels in operation, it's useful to discuss the "ALARMS DIAGRAM" button. This button provides alerts for undesired or erroneous situations in the plant, indicated by red lights below each alarm identifier.

All alarms are of the "Memory" type, with priority levels ranging from 1 to 999, except for the analog alarm, which is of the "Memory Integer" type.

The operation of these panels is controlled by a script coordinating the plant’s schematic diagram. This schematic reflects the real-time status, based on active switches (actuators).





The other panel consists of an indicator showing the container’s loading status according to a previously established color-coded system, already explained in the "ALARM DICTIONARY" window.

The analog alarms are triggered when surpassing a specific threshold (HighHigh); in this case, 90 tons. There is also a previous warning at 50% of this threshold.

The analog alarms can be manually simulated using this panel by adjusting a slider, which allows the user to trigger alarms directly without physically modifying the model, as shown in the figures below.






Window HISTORICAL TREND:

The "HISTORICAL TREND" window is intended to store and keep a record of all alarms triggered over time. It mainly consists of three historical logs.




Two of these historical records (on the left side) store alarm logs in textual form. Both record the day, month, and time when the alarm was triggered, as well as the group it belongs to, the value it reached, the associated comment, and its type.

All this information is reflected in the table corresponding to the nature of the alarm:


  • "ANALOG ALARMS" Table




  • "DISCRETE ALARMS" Table



The other historical record is located on the right side of the window, and its function is to continuously represent the current and past values of the analog variable(s) in the process.

This is achieved through a conventional load-time graph, where the time units can be adjusted between seconds, minutes, hours, or days.
